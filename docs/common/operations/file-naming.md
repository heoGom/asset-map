# 파일 네이밍 규칙

## 목적
프로젝트 내의 모든 문서 파일명을 `kebab-case(-)`로 통일하여 가독성을 높이고, 운영체제 간의 호환성을 확보하며, 자동화 도구 및 검색의 일관성을 유지하는 것을 목적으로 합니다.

## 규칙
- **kebab-case(-)** 사용: 모든 소문자를 사용하고 단어 사이는 하이픈(-)으로 연결합니다.
- **snake_case(_) 금지**: 언더바(_)를 파일명에 사용하지 않습니다.
- **공백 금지**: 파일명에 공백을 포함하지 않습니다.
- **일관성**: `docs` 폴더 내의 모든 마크다운(`.md`) 파일 및 문서 파일에 동일하게 적용합니다.

## 변경 작업 절차
기존에 `snake_case`로 작성된 파일이 발견될 경우 다음 절차에 따라 수정합니다.

1. **파일명 변경**: `snake_case` 파일명을 `kebab-case`로 변환합니다.
2. **내부 링크 수정**: 해당 파일을 참조하고 있는 모든 마크다운 문서 내의 링크를 새 경로에 맞게 수정합니다.
3. **최상위 문서 갱신**: 루트 디렉토리의 `README.md` 및 `AGENTS.md` 등에 포함된 경로를 수정합니다.
4. **검증**: 링크가 깨지지 않았는지, `snake_case` 파일명이 남아있지 않은지 최종 확인합니다.

## 출력 요구사항
작업 완료 후 다음 내용을 요약하여 보고합니다.
- 변경된 파일 목록
- 수정된 링크 목록
