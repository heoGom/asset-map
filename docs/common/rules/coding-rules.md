# 코딩 규칙

## 백엔드 구조

- 기능 기반 패키지(feature-based packages)를 사용하십시오.
- 컨트롤러는 얇게 유지하고 비즈니스 로직은 서비스에 위임하십시오.
- 리포지토리는 영속성 쿼리에만 집중하십시오.
- 요청 및 응답 모델을 엔티티와 분리하여 유지하십시오.
- 컨트롤러에서 엔티티를 직접 노출하지 마십시오.
- 프로젝트 문서에 규칙을 중복 작성하는 대신 `docs/common/*`의 공통 타입을 사용하십시오.

## API 및 DTO 규칙

- 요청 및 응답 페이로드에 DTO를 사용하십시오.
- 문서화된 API 계약은 안정적으로 유지하십시오.
- 응답 형태가 변경되면 클라이언트와 문서를 함께 업데이트하십시오.

## 프론트엔드 UI 규칙

- 모든 프론트엔드 UI는 반응형이어야 합니다.
- 모바일 우선 접근 방식(mobile-first approach)을 사용하십시오.
- UI는 360px 너비에서 올바르게 작동해야 합니다.
- Tailwind 중단점 `sm`, `md`, `lg`를 사용하십시오.
- 버튼, 카드, 목록은 뷰포트가 줄어들 때 자연스럽게 줄바꿈되어야 합니다.
- `overflow`가 레이아웃을 깨뜨리지 않도록 하십시오.
- 과도한 `truncate` 사용보다 텍스트 줄바꿈을 선호하십시오.
- 다크 모드와 라이트 모드 모두에서 텍스트 가독성을 유지하십시오.
- 기존 디자인 시스템 내에서 작업할 때는 기존 동작을 보존하십시오.

## 프론트엔드 레이아웃 규칙

- 모바일에서 터치 대상(touch targets)을 편안하게 유지하십시오.
- 콘텐츠 넘침으로 인한 가로 스크롤을 방지하십시오.
- 간격과 줄바꿈을 조정하여 밀집된 콘텐츠의 가독성을 유지하십시오.
