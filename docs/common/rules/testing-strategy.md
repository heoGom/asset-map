# 테스트 전략

## 핵심 규칙

- HTTP API에 대해 MockMvc 기반의 컨트롤러 및 보안 테스트가 필요합니다.
- 모든 새로운 도메인은 최소한 서비스 테스트와 MockMvc 테스트를 포함해야 합니다.
- 인증이 필요한 API는 JWT를 포함하여 테스트해야 합니다.
- 새로운 기능 및 도메인 변경 시 테스트는 필수입니다.
- 테스트가 없는 기능은 완료된 것으로 간주하지 않습니다.
- 기존 테스트가 실패하면 원인을 분석하고 구현 또는 테스트를 수정하십시오.

## 필수 커버리지

- 성공 케이스
- 권한 부여 실패 케이스
- 검증 실패 케이스
- 소프트 딜리트 영향 케이스

## API 테스트 규칙

- 보안 엔드포인트에 대해 `Authorization` 헤더를 확인하십시오.
- 인증 실패 시 `401`을 확인하십시오.
- 권한 부여 실패 시 `403`을 확인하십시오.
- API가 `ApiResponse<T>`를 반환할 때 공통 응답 구조를 확인하십시오.

## 개발 흐름

1. 기능 설계
2. 기능 구현
3. 서비스 테스트 및 MockMvc 테스트 작성
4. 테스트 실행 및 통과 확인
5. 다음 기능으로 이동
