# API 가이드라인

## 응답 규칙

- 성공 응답은 `ApiResponse<T>`와 같은 공통 응답 래퍼를 사용해야 합니다.
- 에러 응답은 에러 코드와 메시지를 포함하는 안정적인 구조를 사용해야 합니다.
- 컨트롤러는 엔티티를 직접 반환해서는 안 됩니다.
- 응답 형태는 문서화되어야 하며 안정적으로 유지되어야 합니다.

## 인증 규칙

- 보안이 필요한 요청은 `Authorization: Bearer {token}`을 사용해야 합니다.
- 인증 실패 시 `401`을 반환해야 합니다.
- 권한 부여 실패 시 `403`을 반환해야 합니다.
- 인증 실패와 권한 부여 실패는 별도로 처리해야 합니다.

## 검증 규칙

- 잘못된 요청 본문은 `400`을 반환해야 합니다.
- 검증 에러 응답은 예측 가능해야 하며 문서화되어야 합니다.

## 계약 규칙

- API 응답 형태가 변경되면 클라이언트와 문서를 함께 업데이트하십시오.
- 동일한 리소스에 대해 목록, 상세, 변경 응답을 일관되게 유지하십시오.
- 목록 API 전반에서 페이지네이션 및 정렬 파라미터를 일관되게 사용하십시오.
